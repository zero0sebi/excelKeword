# 보안 검토 및 기술 분석 보고서 (Security & Technical Analysis Report)

본 보고서는 **엑셀 키워드 카운터** 프로그램의 데이터 처리 방식과 외부 통신 여부에 대한 보안 검토 결과를 기술합니다.

## 1. 개요 (Overview)
본 프로그램은 사용자가 업로드한 엑셀 파일 내의 키워드를 분석하는 도구로, **서버가 없는(Serverless) 브라우저 기반 애플리케이션**입니다. 모든 데이터 처리는 사용자의 로컬 환경 내에서만 수행되도록 설계되었습니다.

## 2. 데이터 처리 메커니즘 (Data Processing Flow)
1. **파일 읽기**: 브라우저 표준 API인 `FileReader`를 사용하여 로컬 파일 시스템에서 브라우저 메모리로 데이터를 직접 읽어옵니다.
2. **분석 수행**: 메모리에 로드된 엑셀 데이터를 `SheetJS(xlsx.full.min.js)` 라이브러리가 해석하며, JavaScript 루프를 통해 키워드 일치 여부를 판별합니다.
3. **결과 출력**: 분석된 숫자는 즉시 브라우저 화면(DOM)에 렌더링되며, 별도의 파일 저장이 필요한 경우에만 브라우저의 다운로드 기능을 통해 로컬 파일로 생성됩니다.

## 3. 외부 통신 검토 결과 (Network Communication Audit)

| 항목 | 검토 결과 | 상세 내용 |
| :--- | :--- | :--- |
| **데이터 전송 (Outbound)** | **발생하지 않음** | 사용자의 엑셀 내용이나 검색 키워드를 외부 서버로 전송하는 `fetch`, `XMLHttpRequest`, `WebSocket` 등의 통신 코드가 존재하지 않습니다. |
| **추적기/쿠키 (Tracker/Cookie)** | **사용하지 않음** | Google Analytics나 기타 광고/추적용 스크립트가 포함되어 있지 않으며, 쿠키를 생성하지 않습니다. |
| **라이브러리 로드 (Inbound)** | **조건부 발생** | - `_local.html` 버전: 모든 자원을 로컬에서 로드하여 통신 0건.<br>- `_cdn.html` 버전: 최초 실행 시 라이브러리 파일(JS) 1건만 로드함. |

## 4. 라이브러리(`xlsx.full.min.js`) 보안성 확인
- 해당 파일은 세계적으로 널리 사용되는 오픈소스 라이브러리인 **SheetJS(v0.x)**의 배포용 빌드입니다.
- 파일 내부에 포함된 `http://...` 형태의 문자열은 엑셀 파일 형식(OOXML, OpenXML)을 정의하는 **XML 네임스페이스 표준 주소** 및 제작자 홈페이지 주소(주석)일 뿐이며, 실제 네트워크 접속을 유도하는 실행 코드가 아님을 확인하였습니다.

## 5. 보안 검증 방법 (Verification)
보안 담당자 또는 사용자는 다음과 같은 방법으로 본 프로그램의 무결성을 직접 확인할 수 있습니다.

1. **오프라인 테스트**: `keyword_search_local.html`을 실행하기 전 인터넷 연결을 완전히 차단한 상태에서도 모든 기능이 정상적으로 작동함을 확인할 수 있습니다.
2. **네트워크 모니터링**:
   - 브라우저에서 `F12`(개발자 도구)를 누릅니다.
   - **Network(네트워크)** 탭을 선택합니다.
   - 파일 업로드 및 분석 버튼을 클릭했을 때 새로운 네트워크 요청이 한 건도 발생하지 않음을 확인할 수 있습니다.

## 6. 결론
본 프로그램은 사용자의 민감한 데이터(엑셀 내용)를 안전하게 보호하며, 외부로 어떠한 정보도 전송하지 않는 **안전한 도구**임을 확인합니다.
